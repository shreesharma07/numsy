Numsy - Phone Number Validation API

Overview

The Numsy API provides endpoints for uploading and processing CSV/Excel files containing phone numbers. The API validates Indian mobile numbers, removes duplicates, and provides detailed analytics.

Endpoints

POST /api/upload

Upload and process a CSV or Excel file containing phone numbers. Returns processed data with analytics and a download ID.

Request

  • file File (multipart/form-data) Required
    CSV, XLSX, or XLS file (max 50MB)

Response

{
  "success": true,
  "message": "File processed successfully",
  "summary": {
    "totalRecords": 1000,
    "validRecords": 850,
    "invalidRecords": 150,
    "duplicatesRemoved": 50
  },
  "downloadId": "processed_1234567890"
}
GET /api/download/:id

Download the processed ZIP file containing valid numbers, invalid numbers, and analytics report.

Parameters

  • id String (URL parameter) Required
    Download ID returned from the upload endpoint

Response

Returns a ZIP file containing:

  • *_valid_*.csv - Valid phone numbers
  • *_invalid_*.csv - Invalid phone numbers with reasons
  • *_analytics_*.txt - Detailed analytics report
GET /api/health

Health check endpoint to verify API status.

Response

{
  "status": "OK",
  "timestamp": "2026-03-09T12:00:00.000Z"
}

Validation Rules

  • ✓ Must be exactly 10 digits
  • ✓ Must start with 6, 7, 8, or 9
  • ✓ Only valid Indian mobile numbers
  • ✓ Automatic duplicate removal
  • ✓ Support for multiple numbers per row
